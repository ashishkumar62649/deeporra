# DeepOrra v0.1.0 вҖ” Local-First Repository Intelligence

**DeepOrra** is a local-first repository intelligence tool for AI coding agents. It indexes a Python repository on your machine and exposes its structure, symbols, routes, and code relationships through search, a dashboard, and read-only MCP planning tools вҖ” so agents can check what already exists before writing new code.

This is the first public release of DeepOrra.

---

## Key Capabilities

- **Local-first indexing** вҖ” repository index data is stored in `.deeporra/` inside each indexed repository. No code leaves your machine.
- **Python AST analysis** вҖ” extracts functions, classes, methods, imports, and HTTP route definitions (Flask/FastAPI patterns). See `deeporra/parser/`.
- **Semantic embeddings** вҖ” local Sentence Transformers (`all-MiniLM-L6-v2`, 384 dimensions) run on CPU. See `deeporra/embeddings/`.
- **Three search modes** вҖ” lexical (FTS5 full-text), semantic (Chroma vector), and hybrid (combined) ranking. See `deeporra/querying/`.
- **Code graph** вҖ” relationships between files, symbols, imports, routes, and tests stored in SQLite `code_nodes`/`code_edges` tables. See `deeporra/graph/`.
- **Read-only MCP stdio server** вҖ” 8 planning tools for AI coding agents. All tools require a `repository_root` that has been indexed by DeepOrra.
- **Streamlit dashboard** вҖ” localhost-only human-inspection UI for indexed repositories. See `deeporra/dashboard/`.
- **CLI** вҖ” four functional commands (`index`, `status`, `doctor`, `mcp`, `dashboard`) and one deferred stub (`setup`). Entry point: `deeporra` or `python -m deeporra`.

## Quick Start

```bash
# Check your environment
deeporra doctor /path/to/repo

# Index a repository (full rebuild)
deeporra index /path/to/repo

# View index status
deeporra status /path/to/repo
```

If the console script is not on your PATH, use `python -m deeporra` instead of the bare `deeporra` command.

### Starting the MCP Server

Use either the CLI command or the module entry point:

```bash
deeporra mcp --repo /path/to/repo
# or
python -m deeporra.mcp_server
```

Configure your coding agent to launch this command as an MCP stdio subprocess. The server provides 8 read-only planning tools.

### Starting the Dashboard

Use either the CLI command or the module entry point:

```bash
deeporra dashboard --port 8501
# or
python -m deeporra.dashboard
```

Opens a local Streamlit app at `http://localhost:8501` (default port).

---

## MCP Tools (8)

All tools are defined in `deeporra/mcp_server/server.py` and served through a FastMCP stdio server. Every tool accepts `repository_root` (path to an indexed repository) and returns JSON. All tools are read-only.

| Tool | Purpose |
|------|---------|
| `repository_summary` | Return summary statistics for an indexed repository |
| `search_code` | Search code chunks by text, semantic, or hybrid mode |
| `hybrid_search` | Search code using combined text + semantic ranking |
| `find_symbols` | Find functions, classes, and methods matching a name |
| `find_routes` | Find HTTP route definitions by method, path, or handler |
| `get_related_code` | Find related code nodes through graph edges (one hop) |
| `analyze_change_impact` | First-order impact analysis for a symbol |
| `find_existing_implementation` | Search for existing code matching a description |

## Known Limitations

- **Python-only parsing** вҖ” AST support limited to Python. Multi-language parsing (TypeScript, Go, etc.) is deferred.
- **First-order impact only** вҖ” `analyze_change_impact` shows direct relationships, not transitive analysis.
- **No incremental indexing** вҖ” each `deeporra index` run performs a full rebuild.
- **One-hop graph** вҖ” `get_related_code` supports `depth=1` only.
- **No automatic source editing** вҖ” MCP tools are read-only and planning-only.
- **No private repository authentication** вҖ” deferred to a later release.
- **No hosted SaaS** вҖ” local-first only; no cloud deployment is planned.
- **No CI/CD integration** вҖ” deferred to a later release.
- **Windows-tested only** вҖ” primary development and testing on Windows. macOS and Linux not yet validated.

## Upgrade and Compatibility Notice

DeepOrra is in early development (v0.1.0). The indexing pipeline, CLI commands, MCP server, and dashboard are functional, but the API and data format may change in breaking ways before 1.0.0. Test thoroughly before relying on persisted indexes across versions.

There are no previous releases to upgrade from.
